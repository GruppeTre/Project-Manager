# Contributions   
   
We value all contributions, but also must keep the code free of feature creep, scope-creep and with a low as possible technical debt.   
Therefore any pull requests and issues which adds or requests features may not be accepted by the team.
   
## Issues   
First and foremost, please ensure that it is a novel issue, this is to ensure we don't spend time combing through issues which already exist. That's time we could use on making the project more awesome!
Please provide any issues in the following format, yes you can copy-paste it:   

>``# Header: Short descriptive title - max of 50 characters.``  
>``## Type: Is it a bug, a feature or something third``  
>``## Reproducability: For bugs, please include a detailed ordered list of steps to reproduce.``  
>``## Why is this necessary: For feature request, please provide a detailed reason how this would improve the project and why it stays within scope.``
>``## Sign-off: By signing off, you tell us you have looked through open issues first.``
